# Requirements — MuTrade v1.1 Admin Dashboard

*Created: 2026-04-12*

## v1.1 Requirements

### UI — 실시간 모니터링 뷰

- [ ] **UI-01**: 사용자는 브라우저에서 모니터링 중인 종목별 현재가·고점·하락률·임계값을 테이블로 조회할 수 있다
- [ ] **UI-02**: 사용자는 종목 현황이 WebSocket을 통해 자동 갱신되는 것을 확인할 수 있다
- [ ] **UI-03**: 사용자는 드라이런 모드일 때 화면 상단에 경고 배너가 표시되는 것을 볼 수 있다
- [ ] **UI-04**: 사용자는 각 종목별로 현재 상태 배지(모니터링 중 / 매도 신호 / PENDING)를 볼 수 있다
- [ ] **UI-05**: 사용자는 봇 실행 시작 시간과 마지막 가격 업데이트 시각을 확인할 수 있다

### CTRL — 봇 제어 패널

- [ ] **CTRL-01**: 사용자는 버튼 클릭으로 봇을 시작하거나 중지할 수 있다
- [ ] **CTRL-02**: 사용자는 봇 중지 시 확인 다이얼로그를 통해 실수로 중지되는 것을 방지할 수 있다
- [ ] **CTRL-03**: 사용자는 UI에서 모니터링 종목을 추가하거나 제거할 수 있다
- [ ] **CTRL-04**: 사용자는 UI에서 드라이런 모드를 토글할 수 있다

### HIST — 거래 이력

- [ ] **HIST-01**: 사용자는 `[TRADE]` 로그 마커에서 파싱된 매도 이력을 날짜·종목·수량·가격·DRY/LIVE 구분 테이블로 조회할 수 있다

### CFG — 설정 편집기

- [ ] **CFG-01**: 사용자는 config.toml의 종목별 임계값을 UI에서 편집하고 저장할 수 있다
- [ ] **CFG-02**: 사용자는 설정 변경 시 유효성 검증 결과를 즉시 확인할 수 있다

### LOG — 로그 스트림

- [ ] **LOG-01**: 사용자는 logs/mutrade.log의 최신 로그를 브라우저에서 실시간으로 볼 수 있다

## Future Requirements (v1.2+)

- 프로덕션 `tr_id` 검증 및 실거래 end-to-end 테스트
- WebSocket 실시간 시세 수신 (KIS WebSocket으로 폴링 대체)
- 초기 고점을 config에서 수동 지정 (봇 시작 전 매수 종목 지원)
- 다중 사용자 인증 (개인용이므로 현재 불필요)

## Out of Scope

- 주문 실행 UI — 실수 클릭 시 실손 위험, 수동 유지
- 민감 정보(.env) 편집 UI — 보안상 서버 직접 편집 유지
- 실시간 캔들차트 — KIS API 부하 대비 가치 없음
- 자동 매수 — 손실 방어가 핵심 목적
- 복수 증권사 지원 — 한국투자증권 단일 지원 유지
