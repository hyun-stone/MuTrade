<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| APScheduler 스레드 → BotStateHub | 봇 폴링 스레드가 hub.push_snapshot() 및 hub.is_stop_requested() 호출 |
| FastAPI asyncio 루프 → BotStateHub | lifespan/엔드포인트에서 hub.attach_loop(), hub.request_stop() 호출 |
| uvicorn → 외부 네트워크 | 0.0.0.0:8000 바인딩 — 모든 인터페이스에서 접근 가능 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Spoofing | uvicorn 0.0.0.0 바인딩 | mitigate | ARCHITECTURE.md 권고: 127.0.0.1로 제한하거나 SSH 터널 사용. 로컬 개발 환경에서는 0.0.0.0 허용하되, 서버 배포 시 host="127.0.0.1"로 변경할 것을 README에 명시 |
| T-05-07 | Denial of Service | BackgroundScheduler 스레드 leak | mitigate | scheduler.shutdown(wait=False)를 FastAPI lifespan shutdown에서 호출 (create_app의 lifespan yield 후 블록에 scheduler.shutdown() 추가 — Phase 7에서 scheduler를 app에 주입할 때 처리) |
| T-05-08 | Tampering | hub.request_stop() 오남용 | accept | 개인용 봇, 로컬 바인딩. 인증 없이 /api/bot/stop 호출 가능하지만 최악의 경우 봇 폴링 중단 — 금전적 손실 없음 (매도 미발생, 수동 개입 가능) |
| T-05-09 | Information Disclosure | engine.states → push_snapshot 직렬화 | accept | 직렬화 데이터는 code, peak_price, warm만 포함. API 키, 토큰, 계좌번호 미포함. |
| T-05-10 | Denial of Service | time.sleep(poll_interval)이 stop 요청 반응 지연 | accept | 최대 poll_interval(기본 3초) 지연 후 중단. 트레일링 스탑 정확도에 영향 없음. Phase 7에서 필요 시 threading.Event.wait(timeout=poll_interval)로 개선 가능 |
</threat_model>
