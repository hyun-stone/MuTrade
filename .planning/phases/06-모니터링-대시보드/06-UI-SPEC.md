## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 테이블 셀 내 inline padding, 상태 배지 좌우 여백 |
| sm | 8px | 테이블 셀 상하 padding (`padding: 8px 12px`) |
| md | 16px | 섹션 내 요소 간격, 헤더와 테이블 사이 마진 |
| lg | 24px | 컨테이너 좌우 padding |
| xl | 32px | 페이지 상단 여백 |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions: 테이블 셀 좌우 padding은 12px 사용 (`padding: 8px 12px`) — 행 밀도 최적화를 위해 8px 그리드 예외 허용.

## Interaction States

### WebSocket 연결 상태 표시

페이지 상단 우측에 연결 상태 표시줄:

| 상태 | 점(dot) 색상 | 텍스트 | 텍스트 색상 |
|------|-------------|--------|-------------|
| 연결됨 | `#3b82f6` (blue-500) | "실시간 연결" | `#9ca3af` |
| 재연결 중 | `#f59e0b` (amber-500) | "재연결 중..." | `#9ca3af` |
| 연결 끊김 | `#6b7280` (gray-500) | "연결 끊김" | `#9ca3af` |

점(dot) 크기: 8px × 8px, border-radius 50%.

### SELL_PENDING 행 번짙임 애니메이션

```css
@keyframes blink-sell {
  0%, 100% { background-color: #450a0a; }
  50%       { background-color: #7f1d1d; }
}

tr.sell-pending {
  animation: blink-sell 1s ease-in-out infinite;
}

tr.sell-pending td {
  color: #fca5a5;
}

tr.sell-pending .status-badge {
  color: #ef4444;
  font-weight: 600;
}
```

다크 테마에 맞춰 `#fee2e2` / `#fca5a5` (라이트 레드) 대신 `#450a0a` / `#7f1d1d` (다크 레드) 사용.
source: CONTEXT.md D-05의 패턴을 다크 테마에 맞게 조정.

### 하락률 숫자 색상

| 하락률 | 색상 | 의미 |
|--------|------|------|
| 0% ~ -5% 미만 | `#9ca3af` (gray-400) | 정상 범위 |
| -5% ~ -9% 미만 | `#f59e0b` (amber-500) | 경고 범위 |
| -9% 이상 하락 | `#ef4444` (red-500) | 임계 범위 (SELL_PENDING 직전) |

## WebSocket 데이터 계약

JS가 처리하는 메시지 형식 (Python `hub.py` 직렬화 기준):

```json
{
  "005930": {
    "code": "005930",
    "peak_price": 86200.0,
    "current_price": 84500.0,
    "drop_pct": -1.97,
    "warm": true,
    "sell_pending": false
  }
}
```

빈 객체 `{}` 수신 시 → 빈 상태 메시지 렌더링.

`sell_pending: true`인 종목 → `tr.sell-pending` 클래스 적용.

`warm: false`인 종목 → 상태 배지 "워밍업", 하락률 표시 "—".
