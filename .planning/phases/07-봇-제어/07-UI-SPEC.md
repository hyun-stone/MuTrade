## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 아이콘 갭, 배지 내부 패딩 |
| sm | 8px | 버튼 수직 패딩, 셀 수직 패딩 |
| md | 16px | 헤더 수직 패딩, 버튼 수평 패딩 |
| lg | 24px | 헤더 수평 패딩, 컨테이너 수평 패딩 |
| xl | 32px | 컨테이너 상단 마진 |
| 2xl | 48px | 빈 상태 수직 패딩 |
| 3xl | 64px | (미사용 — 레이아웃 범위 초과) |

Exceptions:
- 버튼 터치 타깃 최소 높이: 36px (패딩 8px 상하 + 폰트 20px = 36px 충족)
- 수동 매도 버튼: 셀 내 `padding: 4px 8px` (컴팩트 인라인)
- WebSocket 상태 도트: 8×8px (고정 크기)

Source: 기존 `index.html` 측정값 + 8-point 그리드 정합 확인.

## Interaction Contract

### 시작 버튼 클릭

1. `POST /api/start` 호출
2. 응답 409: 배너 표시 "이미 실행 중입니다"
3. 응답 400 (시장 시간 외): 배너 표시 "시장 시간이 아닙니다 (09:00~15:20 KST)"
4. 응답 200: 배너 없음. WebSocket 스냅샷의 `is_running` 변경으로 버튼 상태 갱신.

Source: CONTEXT.md D-01, D-03.

### 중지 버튼 클릭

1. SELL_PENDING 종목이 있는 경우: `window.confirm("매도 진행 중인 종목이 있습니다. 그래도 중지하시겠습니까?")` 표시
   - 취소: 아무 동작 없음
   - 확인: `POST /api/stop` 호출
2. SELL_PENDING 종목이 없는 경우: 즉시 `POST /api/stop` 호출
3. 응답 200: 배너 없음. WebSocket 스냅샷의 `is_running` 변경으로 버튼 상태 갱신.

Source: CONTEXT.md D-04, D-05, D-06.

### 드라이런 배지 클릭

1. `POST /api/toggle-dry-run` 호출
2. 응답 200: 배너 표시 "드라이런 → 실매도 모드로 전환됨. 재시작 시 .env 설정으로 초기화됩니다" (또는 반대 방향)
3. WebSocket 스냅샷의 `dry_run` 필드 변경으로 배지 자동 갱신.

Source: CONTEXT.md D-07, D-09.

### 즉시 매도 버튼 클릭

1. `window.confirm("[종목코드]을(를) 시장가로 매도하시겠습니까?")` 표시
2. 취소: 아무 동작 없음
3. 확인: `POST /api/sell/{code}` 호출
4. 응답 200: 배너 표시(성공) "매도 주문 제출됨 — 체결 확인 필요"
5. 응답 4xx/5xx: 배너 표시(실패) "매도 실패: {서버 반환 message 필드}"

Source: CONTEXT.md D-10, D-11, D-13.

### WebSocket 스냅샷 수신 시 UI 갱신 항목

| 필드 | 갱신 대상 |
|------|-----------|
| `is_running` | `#btn-start` / `#btn-stop` disabled 상태 |
| `dry_run` | `#dry-run-badge` 텍스트 및 색상 |
| 각 종목 `sell_pending` | 즉시 매도 버튼 disabled + blink-sell 행 스타일 |

## Accessibility Notes

- 모든 버튼은 `<button>` 태그 사용 (클릭 이벤트 + 키보드 Enter/Space 기본 지원)
- disabled 버튼: `disabled` 속성 + `cursor: not-allowed` (스크린리더 제외 신호)
- 배너: `role="alert"` 권장 (스크린리더 즉시 읽기)
- 확인 다이얼로그: `window.confirm()` 사용 (브라우저 네이티브, 접근성 내장)
